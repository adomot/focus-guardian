# デモ動画 台本（3分想定）

撮影構成: 画面収録（アプリ）+ 手元/部屋のカメラ（実演）+ ナレーション。
QuickTime の画面収録 + iPhone で部屋撮りが手軽。編集で 2 画面を並べると伝わりやすい。

## シーン構成

### 0:00–0:20 課題提起（部屋カメラ）
- 机に向かって参考書を開く → 数秒でスマホを手に取ってダラダラ見る演技
- ナレーション: 「勉強するぞと決めたのに、気づけばスマホ。意志力に頼った集中は簡単に崩れます。そこで、AIに見張ってもらうことにしました」

### 0:20–1:00 ヒアリング（画面収録）
- アプリを開き「ヒアリングを始める」
- 目標「美容検定1級を取りたい」→ 悪習慣「スマホをいじってしまう」→ BGM を流す → 集中できるBGM
- 悪習慣2つ目「居眠りをしてしまう」→ 言葉で通知 → 「起きてください」→ もう大丈夫
- ナレーション: 「自由な言葉で伝えた悪習慣を、Gemini が画像で判定できる条件に変換して記憶します」

### 1:00–1:20 セッション開始（画面収録）
- ホーム → 設定内容が表示されている → 「セッション開始」→ カメラ許可 → ダッシュボード表示（監視中インジケータ・状態: 集中中）
- ナレーション: 「あとは作業するだけ。60秒ごとにWebカメラの画像をエージェントが自律判定します」
- ※撮影時は `?interval=10` を付けて判定間隔を10秒に短縮すると待ちが減る

### 1:20–2:10 検知と介入（2画面: 部屋カメラ + 画面収録）
- 机でスマホを触り始める → ダッシュボードが「サボり検知: スマートフォンを操作しています」に変わる → **Echo から BGM が流れ出す**（ここが最大の見せ場。Echo にカメラを寄せる）
- スマホを置いて作業に戻る → ダッシュボードが「集中中」に戻る
- （可能なら）居眠りの演技 → Echo から「起きてください」
- ナレーション: 「検知は連続2回・確信度つき。介入は1回だけ、しつこくしません。介入後に作業へ戻れたかまでエージェントが評価します」

### 2:10–2:40 サマリーとアーキテクチャ（画面収録 + 図）
- セッション終了 → サマリー画面（集中時間・検知回数・介入と復帰数）
- architecture.png を表示
- ナレーション: 「Cloud Run 上の ADK エージェントが Gemini で判定し、Voice Monkey 経由で Alexa へ。画像はどこにも保存しないプライバシー設計。GitHub Actions でテストからデプロイまで自動化しています」

### 2:40–3:00 クロージング（部屋カメラ）
- 集中して勉強している姿
- ナレーション: 「監視対象はあなたの言葉で決まる、あなた専属の集中パートナー。Focus Guardian でした」

## 撮影チェックリスト

- [ ] Echo の音量を十分に上げる（動画に音が入るように）
- [ ] 部屋の照明を明るく（Gemini の判定精度・映像品質の両方に効く)
- [ ] デモ用に判定間隔を短縮（`?interval=10`）
- [ ] introの演技はマスク/顔出しの許容範囲を事前に決める（顔出しNGなら後ろから撮る）
- [ ] 万一 Voice Monkey が不調な場合はブラウザ再生のフォールバックで撮り、その旨をナレーションで説明
